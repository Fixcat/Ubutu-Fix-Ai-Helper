# Руководство по внесению вклада в FixAdm

Спасибо за интерес к проекту FixAdm! Мы рады любому вкладу.

## Как внести вклад

### Сообщения об ошибках

Если вы нашли ошибку:
1. Проверьте, не была ли она уже сообщена в [Issues](https://github.com/Fixcat/Ubutu-Fix-Ai-Helper/issues)
2. Создайте новый Issue с подробным описанием:
   - Версия FixAdm
   - Версия Ubuntu
   - Используемый AI провайдер
   - Шаги для воспроизведения
   - Ожидаемое и фактическое поведение
   - Логи ошибок

### Предложения новых функций

1. Откройте Issue с тегом "enhancement"
2. Опишите предлагаемую функцию
3. Объясните, почему она будет полезна
4. Приведите примеры использования

### Pull Requests

1. Форкните репозиторий
2. Создайте ветку для вашей функции (`git checkout -b feature/amazing-feature`)
3. Внесите изменения
4. Протестируйте изменения
5. Закоммитьте (`git commit -m 'Add amazing feature'`)
6. Запушьте в ветку (`git push origin feature/amazing-feature`)
7. Откройте Pull Request

### Стандарты кода

- Используйте bash для скриптов
- Следуйте существующему стилю кода
- Добавляйте комментарии для сложной логики
- Проверяйте синтаксис: `bash -n script.sh`

### Тестирование

Перед отправкой PR убедитесь что:
- Скрипт запускается без ошибок
- Все зависимости указаны
- Документация обновлена
- Примеры работают

### Документация

При добавлении новых функций:
- Обновите README.md
- Добавьте примеры в EXAMPLES.md
- Обновите CHANGELOG.md
- Добавьте комментарии в код

## Стиль коммитов

Используйте понятные сообщения коммитов:
- `feat: добавлена поддержка новой модели`
- `fix: исправлена ошибка с сохранением конфигурации`
- `docs: обновлена документация`
- `refactor: улучшена структура кода`
- `test: добавлены тесты`

## Вопросы?

Если у вас есть вопросы, создайте Issue с тегом "question".

Спасибо за ваш вклад! 🚀
