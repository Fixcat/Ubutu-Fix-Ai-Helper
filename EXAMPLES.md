# Примеры использования FixAdm

FixAdm - это AI-powered администратор для Ubuntu, который помнит контекст разговора (последние 50 сообщений) и может выполнять сложные задачи администрирования.

## Работа с памятью

FixAdm помнит последние 50 сообщений, что позволяет вести естественный диалог:

```
Вы: установи nginx
FixAdm: [устанавливает nginx]

Вы: теперь настрой его для работы с PHP
FixAdm: [настраивает PHP-FPM и интеграцию с nginx]

Вы: создай виртуальный хост для mysite.com
FixAdm: [создает конфигурацию виртуального хоста]

Вы: добавь SSL сертификат
FixAdm: [устанавливает certbot и получает сертификат]
```

Используйте `/history` чтобы узнать сколько сообщений в памяти, и `/reset` чтобы начать новый разговор.

1. Установку Docker (если не установлен)
2. Клонирование репозитория Marzban
3. Настройку docker-compose
4. Настройку firewall (открытие портов)
5. Запуск сервиса
6. Вывод информации для доступа

## Советы по использованию

1. **Будьте конкретны**: Чем точнее запрос, тем лучше результат
2. **Проверяйте команды**: Всегда читайте что FixAdm предлагает выполнить
3. **Используйте контекст**: FixAdm помнит последние 50 сообщений
4. **Задавайте вопросы**: Если что-то непонятно, спросите FixAdm объяснить
5. **Используйте /reset**: Если разговор ушел не в ту сторону, очистите память
6. **Редактируйте команды**: Нажмите 'e' при подтверждении чтобы отредактировать команду
7. **Используйте /settings**: Меняйте провайдера или модель прямо из чата без перезапуска

## Примеры диалогов

### Диалог 1: Установка веб-приложения
```
Вы: мне нужно развернуть Laravel приложение
FixAdm: Хорошо, для Laravel нужны: nginx, php-fpm, composer, mysql. Начнем с установки?
Вы: да
FixAdm: [выполняет команды установки]
Вы: теперь настрой виртуальный хост для myapp.local
FixAdm: [настраивает nginx]
```

### Диалог 2: Решение проблемы
```
Вы: сайт не открывается, ошибка 502
FixAdm: Проверю статус nginx и php-fpm
FixAdm: [выполняет диагностику]
FixAdm: Проблема в php-fpm, он не запущен. Запустить?
Вы: да
FixAdm: [запускает сервис]
FixAdm: Сервис запущен, проверьте сайт
```

### Диалог 3: Использование памяти
```
Вы: установи docker
FixAdm: [устанавливает docker]

Вы: добавь меня в группу docker
FixAdm: [добавляет пользователя в группу]

Вы: теперь установи portainer
FixAdm: [устанавливает portainer через docker]
# FixAdm помнит что docker уже установлен и не устанавливает его заново
```

### Диалог 4: Изменение настроек
```
Вы: /settings
FixAdm: [показывает меню настроек]
Вы: 1 (выбираем смену провайдера)
FixAdm: [показывает список провайдеров]
Вы: 2 (выбираем Claude)
FixAdm: ✓ Провайдер изменен на: Claude
FixAdm: История разговора очищена (новый провайдер)
Вы: [продолжаем работу с новым провайдером]
```

## Ограничения

- FixAdm не может выполнять команды требующие интерактивного ввода
- Некоторые операции могут требовать перезагрузки
- Всегда делайте бэкапы перед серьезными изменениями
- Память ограничена 50 сообщениями (используйте /reset для очистки)
