# Политика безопасности

## Поддерживаемые версии

| Версия | Поддержка         |
| ------ | ----------------- |
| 1.1.x  | ✅ Поддерживается |
| 1.0.x  | ⚠️ Ограниченная   |
| < 1.0  | ❌ Не поддерживается |

## Рекомендации по безопасности

### API ключи

- **НИКОГДА** не коммитьте API ключи в репозиторий
- Храните ключи в `~/.fixadm-config` (права 600)
- Используйте разные ключи для разработки и продакшена
- Регулярно ротируйте ключи

### Выполнение команд

- **ВСЕГДА** проверяйте команды перед выполнением
- Используйте опцию редактирования (e) для изменения команд
- Не запускайте FixAdm с правами root без необходимости
- Делайте бэкапы перед серьезными изменениями

### Конфигурационные файлы

- Проверяйте права доступа к конфигам
- Не храните чувствительные данные в открытом виде
- Используйте sudo только когда необходимо

### Сетевая безопасность

- API запросы идут через HTTPS
- Не используйте FixAdm в недоверенных сетях без VPN
- Проверяйте сертификаты API провайдеров

## Известные ограничения

1. API ключи хранятся локально в открытом виде (планируется шифрование)
2. История разговоров хранится без шифрования
3. Команды выполняются с правами текущего пользователя

## Лучшие практики

1. Используйте ограниченные API ключи (только необходимые права)
2. Регулярно проверяйте историю выполненных команд
3. Используйте `/reset` для очистки чувствительных данных из памяти
4. Не используйте FixAdm для критических production систем без тестирования
5. Делайте снапшоты системы перед масштабными изменениями
